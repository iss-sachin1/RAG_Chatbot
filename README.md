# RAG Chatbot

A Retrieval-Augmented Generation chatbot built with Next.js. Upload a PDF (or
image), it gets chunked and indexed into a vector store, and you can chat with
your documents. Answers are generated by **NVIDIA-hosted MiniMax-M3**; retrieval
is powered by **Upstash Vector** (with server-side embeddings).

## Architecture

| Concern        | Service                                   |
| -------------- | ----------------------------------------- |
| LLM            | MiniMax-M3 via NVIDIA (OpenAI-compatible) |
| Vector store   | Upstash Vector (built-in embeddings)      |
| PDF parsing    | LangChain `WebPDFLoader` (pdfjs)          |
| Image OCR      | `tesseract.js`                            |
| Web framework  | Next.js (App Router, route handlers)      |

The vector store and LLM are both HTTP services, so the app runs on serverless
platforms (Vercel) with no separate database server to host.

## Getting Started

### 1. Configure environment

Copy `.env.example` to `.env` and fill in the values:

```bash
cp .env.example .env
```

- `NVIDIA_API_KEY` — from https://build.nvidia.com/minimaxai/minimax-m3
- `UPSTASH_VECTOR_REST_URL` / `UPSTASH_VECTOR_REST_TOKEN` — create an index at
  https://console.upstash.com/vector **with a built-in embedding model**
  (e.g. `mixedbread-ai/mxbai-embed-large-v1`), then copy its REST URL and token.

### 2. Run the dev server

```bash
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

You can start editing the page by modifying `app/page.tsx`. The page auto-updates as you edit the file.

This project uses [`next/font`](https://nextjs.org/docs/app/building-your-application/optimizing/fonts) to automatically optimize and load [Geist](https://vercel.com/font), a new font family for Vercel.

## Deploy on Vercel

1. Push this repo to GitHub (already done if you're reading this on GitHub).
2. In [Vercel](https://vercel.com/new), import the repository. Next.js is
   auto-detected — no build configuration needed.
3. Add the environment variables from `.env.example` under
   **Project Settings → Environment Variables**:
   - `NVIDIA_API_KEY`, `NVIDIA_BASE_URL`, `NVIDIA_MODEL`
   - `UPSTASH_VECTOR_REST_URL`, `UPSTASH_VECTOR_REST_TOKEN`
4. Deploy.

> **Note on OCR:** image OCR uses `tesseract.js`, which is heavy in serverless
> environments. PDF upload/chat is the primary supported path on Vercel; image
> OCR may need a longer function timeout or a dedicated worker.

The API routes (`/api/upload`, `/api/chat`) run on the Node.js runtime with a
60-second `maxDuration` to accommodate parsing and embedding.
